用户明确要求'别着急自己开发，先找好的技能'——在动手做视觉/动画改造前，先花时间搜寻、对比、安装高质量的 iOS/SpriteKit 相关 Agent 技能。筛选标准：GitHub star 数和仓库活跃度是第一参考，低 star（<20）的技能不值得安装。搜索渠道：GitHub topics、技能排行榜站、社区维护者的 Agent 技能索引、社区 awesome 列表。为什么：用户认为 agent 直接手写代码容易产出'太差'的结果，而有高 star 技能指导能一次性做对。怎么用：收到 iOS 开发类任务后，先搜 GitHub 上 star>50 的 SKILL.md 仓库 + 索引站，安装 2-4 个最优质技能，再基于技能指导动手；低星技能果断跳过。
