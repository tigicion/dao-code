# i18n 主循环 UI 全量 实现计划(B 续:做全英文体验)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** 把主循环运行时 UI 的硬编码中文(`App.tsx` ~180 串 + `types.ts` + `index.ts` 迁移提示)全部按 B 的 i18n 模式跟随 locale,使英文用户进入对话、敲 `/help` 等也是英文。

**Architecture:** 沿用 B 的 `src/i18n/`(`t()` + zh/en 扁平字典 + `zh/en 键集对称` parity 护栏)。把目标文件的中文串抽成带命名空间的键(zh 值逐字抄源、en 值翻译),原位换成 `t(key)`。不引入新机制。

**Tech Stack:** TS ESM、React+Ink、vitest+ink-testing-library、B 的 i18n。

**前序:** B(i18n 展示层)已合并;`t/setLang/getLang` 与 zh/en 字典就绪;`i18n.test.ts` 有 `zh/en 键集对称` 结构护栏(新增键必须 zh/en 同步)。

## Global Constraints

- **zh 值逐字抄现有源码串**(含全角标点/emoji 前缀如 `✎ ⊙ ◇ ※ ∞ ↩ ✓`);**en 值自然地道翻译**,保留同样的符号前缀与占位 `${…}`/`{0}` 结构。
- **新增键必须同时进 `zh.ts` 与 `en.ts`**(parity 护栏会 fail 单边)。键命名用命名空间:`mode.*`(权限模式)、`cmd.<命令名>`(斜杠命令描述)、`ui.*`(运行时提示/通知/状态栏/工具标签)。
- **不动**:注释;`builtin.ts` 的命令 prompt(model-facing,另算);道家名句/太极/词标;工具名(`read_file` 等);命令名本身(只译描述)。
- 含变量的串用位置占位 `{0}{1}`(如 `已添加并切到「{0}」` → en `Added and switched to "{0}"`)。
- 每任务跑该区域 focused 测试 + `i18n.test.ts`(parity)+ `npm run typecheck`;ESM `.js` 后缀;commit 不加 AI 署名。

## 翻译质量基线(所有任务遵循)

- 术语统一:权限模式 permission mode;子代理 subagent;检查点 checkpoint;回合 turn;上下文 context;打断 interrupt;审批 approval;技能 skill;记忆 memory。
- 命令描述风格:祈使句、精炼一行,对标 CC 命令面板(如 `查看后台任务` → `Show background tasks`)。
- 保留所有 `/命令`、`@`、`Shift+Tab`、`Esc` 等字面 token 不译。
