# 案例研究:从零造一门语言解释器,¥0.16

> 2026-06-25 · dao v0.1.20

`dao --goal` 全自主、零人工,用 Rust 标准库从零写出一门小语言(MiniLang)的完整解释器
(728 行 / 4 模块)。在 **4000+ 个它从没见过的随机程序上,与权威参考实现逐字节一致**。
总成本 **¥0.164(≈ $0.023)** —— 一次完整的"造语言"任务,成本两分钱。

## 任务

实现 MiniLang 解释器(词法 + 语法 + 求值):`int`/`bool` 两种类型、`let`/`print`/`if-else`/`while`、
floor 除法与取模、短路布尔、类型检查、单一全局作用域。只用 Rust 标准库。

## 功能符合要求(不可糊弄)

差分测试:把 dao 的解释器输出与一个权威参考解释器逐字节比对。dao 开发时只对着「种子 1234 / 400 用例」迭代;
验收后换两个全新种子各跑 2000 个**它没见过的随机程序**:

| 验证 | 用例数 | 结果 |
|---|---|---|
| 原种子 1234 | 418 | **418/418** |
| 新种子 99999 | 2018 | **2018/2018** |
| 新种子 7 | 2018 | **2018/2018** |
| 负对照(`cat` 只回显) | — | 0/48(验收有牙) |

→ 4000+ 个未见程序全部一致 = 真的实现对了,不是背答案。手写 golden 覆盖 floor 除法负数、短路、优先级、错误用例。

## 成本(两路交叉验证)

| 指标 | 值 |
|---|---|
| 总成本 | **¥0.164(≈ $0.023)** |
| 输入 token | 473,572(缓存命中 441,216) |
| 输出 token | 10,442 |
| 缓存命中率 | **93.2%**(dao 自报)== 93.2%(wire 层日志代理独立测得) |
| API 调用 / 墙钟 | 20 次 / 3.3 分钟 |

成本之所以低,是吃满 DeepSeek 前缀缓存(命中价 ≈ 未命中的 1/10):dao 请求前缀逐字节稳定,93% 输入走缓存价。

## 过程

全程 `--goal` 自主:建 Cargo 工程 → 写 lexer/parser/eval/main 四模块 → 撞上 2 个 Rust 借用检查/闭包编译错 →
**自我诊断并修复** → 编译通过 → 跑差分 → 418/418 一次过。零人工干预。

## 复现

```bash
DEEPSEEK_API_KEY=... dao --goal "按 DAO.md 实现 MiniLang 解释器,跑通 python3 tests/diff_test.py"
```
